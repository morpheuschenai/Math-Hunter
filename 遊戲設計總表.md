## 1. 概念階梯（關卡組 × 數學概念 × 新玩法）

每個「關卡組」= 一個新數學概念 + 一個新玩法 + 一次強制教學對話框。
**設計原則：先用「拉長鏈極大化攻擊」的上癮鉤子穩住人，再逐層加運算與數性。**

| 關卡組 | 區域 | 數學概念 | 新玩法 / 新珠 | 教學 id | 狀態 |
|---|---|---|---|---|---|
| 1 | 🌿 湊十草原 | 加法、數感 | 基礎：血量、攻擊倒數、**長鏈＝高傷** | （開場） | ✅ 已上線 |
| 2 | 🌲 進位森林 | 減法 | −珠「先超過再減回來」 | `minus` | ✅ 已上線 |
| 3 | ✖️ 乘法谷 | **乘法** | 同數共鳴（連相同數字=乘法）+ ×2 倍率珠 | `resonance`/`mul` | ✅ 已上線 v10 |
| 4 | ➗ 除法谷 | **除法 / 等分** | 等分挑戰（把目標分成 N 等份）| `divShare` | ✅ 已上線 v10 |

### 除法谷機制細節（v10 改版：靈活、可規劃）

> 改版原因：原「等分盾」目標÷份數只有**唯一解**（18÷2 只能兩顆 9），太僵硬。改為兩種靈活機制，與乘法（×珠＋共鳴）對稱。

- **整除盾 `divisor`（boss 版 `bossDivisor`）｜主力**：目標改為「**湊出 N 的倍數**」——總和能被 N 整除即可（N、2N、3N… 都行），**超多解**。教倍數/整除性。`chainValid` 用 `sum % N === 0`；HUD 顯示「N的倍數」；solver 以 `divisor` 支援（停用 sum 邊界剪枝）。boss 狂暴後 N+1。
- **÷珠（除法珠）｜進階**：藍色 ÷2/÷3 珠（珠值 `-102/-103`），**後置**運算把「目前累積總和」除下去，且**只有整除才連得上**（canAppend 擋）。「先堆一個大數再除到目標」→ 同一目標多種路徑。`chainCalc` 後置除、`badDiv` 防呆；solver 在 DFS 處理 ÷（須 `sum%d===0`、停用 sum 剪枝）。`divRate` 控制出現率、`intro:'div'` 教學。
- **等分盾 `divShare`（保留為入門）**：僅 e1 第一隻當概念示範（把目標分成 N 等份＝剛好 N 顆相同），其餘改用整除盾。
- 關卡：e1 等分盾入門→整除盾(2/3 的倍數)；e2 ÷珠登場(先堆再除命中固定目標)；e3 整除盾+÷珠混合+霸主 bossDivisor。
- 驗證：整除盾 N2~5、÷珠+固定目標、混合，跨 DDA 共 2,000+ 局 100% 有解；解路徑回查 ÷珠（整數命中）與整除盾（sum 為 N 倍數）皆 0 違規。

### 乘法谷機制細節（v10）

- **同數共鳴**：一條鏈裡連到「最長 R 顆相同數字」(R≥2) → 視為乘法 `值×R`，傷害 ×(1 + 0.5×(R−1))。不改變總和(4+4+4 本來就=12)，所以 solver/核心零風險。首次觸發跳 `resonance` 教學。`chainCalc().maxRun` 提供。
- **×2 倍率珠**：珠值 `v=-2`(負值=倍率珠，倍數=−v)。鏈穿過後「下一顆數字×2」再加總。規則同減法珠：不可開頭/結尾、不可緊接另一顆運算珠。`mulRate` 控制出現率。首次跳 `mul` 教學。
- **共鳴盾 `weakRun`**：需鏈中連到 ≥n 顆相同數字才破盾（讓共鳴變必要）。solver 以 `needRun` 支援。
| 5+ | 🕳️ 封印洞窟 / 🌋 烈焰火山 | 質數·奇偶·倍數 | 數性珠 + 隊長 / 敵人弱點 | 各 gimmick | ⏳ 部分上線（奇數盾） |

> **重要規則：數性玩法只鎖「路徑用了哪些珠」，絕不鎖「總和的性質」。**
> 因為目標數字是固定的，總和已被鎖死，再要求「總和為偶數/質數」會自相矛盾。
> 所有隊長技與敵人弱點都建立在「路徑組成」之上（見 §3、§5）。

### 規劃中的新弱點（沿用「路徑組成」框架，皆需加 solver 支援）

| 名稱 | 規則 | 教哪個概念 |
|---|---|---|
| 偶數盾 `weakEven` | 鏈中需含 ≥ n 顆偶數珠 | 奇偶 |
| 質數盾 `weakPrime` | 鏈中需含 ≥ n 顆質數珠（2·3·5·7）| 質數 |

## 5. 隊長技表 `CAPTAIN_DEFS`（常駐，靠路徑組成發動）

**出戰第一隻＝👑隊長**，其隊長技整場常駐。**選隊長＝選一種數學打法**，這是構築深度的來源，也是教育與遊戲的合流點。
所有效果只讀「路徑組成」，不碰總和。

| multi | ⚡ | 倍數爆擊 | 鏈長為 3 的倍數時 傷害 ×2 | 倍數感 |

| 系 | 名稱 | 效果 | 教什麼 |

## 7. 經濟與掉落

### 7a. 現況（v8 已實作）

**蛋掉落 `rollEgg()`**
- 首次過關：**必掉 1 顆蛋**。
- 重玩同關：**35% 機率**掉 1 顆。
- 蛋池：標準模式＝`pool ≤ 該關 zone`（zone 0–3，越後面池越大）；junior＝`pool ≤ 1`。池內**均勻隨機**。

**孵化與升級**
- 孵到新數靈 → 加入圖鑑、可出戰。
- 孵到重複 → **該數靈升級**（`COLL[id]++`），提升基礎傷害。

**基礎傷害 `baseDmg()`** = `12 + Σ(每隻出戰夥伴 (等級−1)) × 2`
**單鏈傷害** = `baseDmg × (鏈長 + 隊長addLen) × 隊長mult × 幸運`

**星星門檻**

| 模式 | 1★ | 2★ | 3★ |
|---|---|---|---|
| 標準 | 過關 | 平均鏈長 ≥ 3 | 平均 ≥ 3 且 最長鏈 ≥ 5 |
| junior | 過關 | 平均 ≥ 2.5 | 平均 ≥ 2.5 且 最長 ≥ 4 |

### 7b. 星星數 = 抽蛋數（point 1）✅ 已上線 v9，v10 改為「新星星」制

讓「拿三星」有實質誘因。實作 `rollEggs(prevStars, stars)` + 多蛋佇列。
**規則：依「這次新拿到的星星數」發蛋**（`gain = max(0, stars − prevStars)`，每顆新星 1 顆蛋）。

| 情境 | 發蛋數 |
|---|---|
| 首次過關 3★ | 3 |
| 首次 1★，之後進步到 3★ | 1 → 再補 2（累計 3）|
| 已 3★ 重玩 | 30% 機率 1 顆（安慰蛋）|

> 為何改：舊版只有「第一次過關」依星數發蛋，導致「先低星過關、之後才拿到 3★」補不到蛋。
> 改成「新星星制」後，湊滿 3★ 一定累計拿到 3 顆，且獎勵「進步」。

### 7b-4. 抽蛋券 + 孵化所頁面（v10 UX 改版）✅

把「即時孵化」改成「**抽蛋券（DRAWS）+ 專屬孵化所頁面**」，提升抽蛋儀式感與重要性。

- **過關結算**：不再當場孵蛋，改顯示「🥚 獲得 N 次抽蛋機會！（共 X 次）」+「前往孵化所抽蛋」按鈕。券數累積存 `nh5_draws`。
- **孵化所頁（`gachaOverlay`）**：顯示券數，點「抽一次 🥚」消耗 1 券 → 蛋→角色的孵化演出；可連抽到券用完。地圖「🥚 孵化所 (N)」按鈕隨時可進。
- **抽蛋池**：`curMaxPool()` 依玩家已破關的最高區域決定（越深入池越大）；junior 固定 pool≤1。
- 遙測：`draws_earned`（earned/total/stars）、`hatch`(source:'gacha')。匯出加 `draws`。
- 未來無盡塔（§7c）的抽蛋獎勵直接累加到同一個 DRAWS。
- 之後可疊「稀有度」：3★ 提高稀有蛋（金蛋）機率（見 §7d）。

### 7b-2. 隊長選擇（point 2）✅ 已上線 v9

隊長**不再是全域固定**。點關卡後若出戰 ≥2 隻，跳「這關誰當隊長？」選擇框，選定者設為 `EQUIP[0]`。只帶 1 隻則直接開戰。也可在「我的數靈」頁設定預設順序。

### 7b-3. 測試開關：全關卡解鎖 ✅ 已上線 v9

地圖「🔓 全解鎖」按鈕，`nh5_unlockall` 持久化，**測試版預設開**。關閉則恢復循序解鎖（前一關 ≥1★）。清除資料時刻意保留此開關。

### 7c. 無盡塔 + 攻擊力分段抽蛋（point 2）✅ 已上線 v10

地圖「🗼 無限塔」進入。每層 1 隻怪，HP/atk 隨層數遞增、band 放大、3F 起輪流出現硬殼/封印/奇數盾/共鳴盾、4F 起減法珠、7F 起 ×2 珠。
- **每過一層 → 🥚 +1 券**（立即入帳 DRAWS）。
- **撤退（↩︎ 離開）或被擊倒 → 結算**：依本局最高單發攻擊力 `towerPeak` 分段加碼送券（0→0、≤100→1、≤1000→2、≤5000→3、≤20000→4、>20000→5）。
- solver 已驗證 1~20F（含各 gimmick）100% 有解。用最高「單發」攻擊力鼓勵長鏈×隊長×共鳴的極大化操作。
- 遙測：`tower_start` / `tower_floor`(floor,peak) / `tower_end`(floors,peak,floor_draws,bonus,died)。

### 7c-old. 原規劃（保留參考）🔜

**目標**：給已破關玩家的終局內容；用「衝高單發攻擊力」當新的上癮鉤子，同時是 DDA 的最佳試驗場。

- **形式**：單一無盡關卡，每層 1 隻怪，HP/atk 隨層數遞增（不設目標 band 上限，難度持續爬升）。
- **每過一層**：1 次抽蛋機會。
- **單局結束額外抽蛋**：依本局**最高單發攻擊力（爆發傷害）**分段給：

| 最高單發攻擊力 | 額外抽蛋 |
|---|---|
| 1 – 100 | 1 |
| 101 – 1,000 | 2 |
| 1,001 – 5,000 | 3 |
| 5,001 – 20,000 | 4 |
| 20,001 + | 5 |

> 用「單發攻擊力」而非總傷，能直接獎勵「長鏈 × 隊長技」的極大化操作，呼應核心玩法。

## 8. DDA 動態難度參數

場內自適應：每隻怪結束後依表現微調目標數帶寬偏移（隱形，玩家不應察覺）。

| 參數 | 標準 | junior |
|---|---|---|
| 偏移上下界 | −4 ～ +10 | −3 ～ 0（只放寬不加難）|
| 輕鬆過（失敗率≤25%、0 提示）| +1 | 0 |
| 掙扎（失敗率≥55% 或 提示≥2）| −1 | −1 |
| 整關失敗 | −2 | −2 |

> band 套用：`hi += off；lo += round(off×0.4)`，下限不低於標準 6 / junior 4，且 `hi ≥ lo+2`。solver 保證仍有解。

## 9. 遙測事件 schema

`log(type, props)` → 存 localStorage `nh5_events`，選關畫面可一鍵匯出 JSON。用 `analyze.py` 分析。

| 事件 | 主要欄位 |
|---|---|
| `session_start` | version |
| `stage_start` | stage, equip, **captain** |
| `first_move` | stage, foe, latency_ms |
| `chain` | len, sum, target, valid, reason, used_minus, **used_mul, run**, odds, primes, cap, cap_mult, cap_addlen, need_odd, **need_run**, minLen, banned, drag_ms |
| `hint` | stage |
| `skill` | skill, stage, foe |
| `enemy_kill` / `boss_enrage` | stage, foe / kind |
| `stage_clear` | stars, dur_ms, avg_chain, max_chain, chains, minus_chains |
| `stage_fail` / `stage_abort` | stage, foe, dur_ms |
| `egg_drop` / `hatch` / `equip` | species / dup, level / team, captain |
| （存檔）`nh5_unlockall` | 測試用全解鎖開關 |
| `dda` | offset, delta, fail_rate, reason |
| `teach` | id, stage |

`reason` 取值：`ok` / `sum`（湊錯數）/ `short`（鏈太短）/ `odd_shield` / `run_shield`（共鳴盾未滿足）/ `end_op`（以運算珠 −／× 結尾）。
